# Contributing to RingSathi

First off, thank you for considering contributing to RingSathi! It's people like you that make RingSathi such a great tool.

## General Guidelines

1. **Fork the repository** to your own GitHub account.
2. **Clone the project** to your local machine.
3. **Create a new branch** for your feature or bug fix (`git checkout -b feature/your-feature-name` or `git checkout -b fix/your-bug-fix`).
4. **Make your changes**, writing descriptive commit messages.
5. **Run tests & linting** locally to make sure nothing is broken.
6. **Push your changes** to your fork.
7. **Submit a Pull Request** to the `main` branch of the original repository.

## Coding Standards

- We use **TypeScript** for everything. Ensure you include proper types for your additions.
- We use **Next.js App Router** conventions. Familiarize yourself with the folder structure.
- Run `npm run lint` before committing to ensure code style consistency.
- Keep components modular. If a component is growing too large, consider breaking it down.

## Bug Reports and Feature Requests

Please use GitHub issues to report bugs or request new features. Make sure to provide detailed steps to reproduce bugs, including the environment (browser, OS, device).

Thank you for your contributions!
